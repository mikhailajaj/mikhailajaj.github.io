
# Mikhail Ajaj Portfolio - Developer Guide

**Professional Portfolio** showcasing multi-domain technical expertise with **$30M+ demonstrated business impact** across five specialized domains.

- **Framework**: Next.js 15.0+ with App Router
- **Language**: TypeScript 5.0+ (strict mode)
- **Styling**: Tailwind CSS 3.4+ with custom design system
- **Animation**: Framer Motion 11.11+
- **3D Graphics**: React Three Fiber + Drei
- **Testing**: Jest + React Testing Library
- **Deployment**: Vercel/GitHub Pages (static export)

- Multi-domain expertise showcase (5 specialized domains)
- Interactive 3D visualizations and animations
- Comprehensive error tracking and analytics system
- WCAG 2.1 AA accessibility compliance
- Progressive Web App capabilities
- Real-time performance monitoring
- Domain-specific theming and routing

## Development Guidelines

### Code Organization
- **Feature-first structure**: Group related components, hooks, and utilities
- **Atomic design principles**: Build from atoms → molecules → organisms
- **Domain-driven design**: Separate concerns by business domain
- **Separation of concerns**: Clear boundaries between UI, logic, and data

### TypeScript Standards
- **Strict mode enabled**: All code must pass TypeScript strict checks
- **Explicit typing**: Avoid `any`, use proper type definitions
- **Interface over type**: Use interfaces for object shapes
- **Path aliases**: Use `@/` for absolute imports from project root

### Component Conventions
- **Naming**: Use PascalCase for components, descriptive names
- **Props**: Define explicit interfaces, avoid generic props
- **Exports**: Use named exports, default export for main component
- **Documentation**: Include JSDoc comments for complex components

### Component Naming Patterns
- **Interactive*** - Components with user interaction and animations
- **Animated*** - Components with motion and visual effects
- **Domain*** - Components with domain-specific functionality
- **Multi*** - Components with multiple steps or states
- **Video*** - Components with video capabilities
- **Business*** - Components focused on business metrics
- **HCI*** - Components following Human-Computer Interaction principles

### Import Structure
```typescript
// 1. React Imports
import React, { useState, useEffect } from 'react';

// 2. External Libraries
import { motion } from 'framer-motion';
import { FaIcon } from 'react-icons/fa';

// 3. Internal Absolute Imports (@/)
import { Component } from '@/components/ui/Component';
import { useHook } from '@/hooks/useHook';

// 4. Internal Relative Imports
import './Component.css';

// 5. Type Imports
import type { ComponentProps } from './types';
```

## Performance Standards

### Core Web Vitals Targets
- **First Contentful Paint (FCP)**: < 1.5s
- **Largest Contentful Paint (LCP)**: < 2.5s
- **Cumulative Layout Shift (CLS)**: < 0.1
- **First Input Delay (FID)**: < 100ms

### Bundle Size Limits
- **Core framework**: < 100KB gzipped
- **Domain pages**: < 50KB additional per domain
- **Component chunks**: < 20KB per lazy-loaded component

### Optimization Techniques
- **Code splitting**: Lazy load components and routes
- **Image optimization**: Next.js Image component with proper sizing
- **Font optimization**: Progressive loading with fallbacks
- **Tree shaking**: Import only needed modules

## Accessibility Requirements

### WCAG 2.1 AA Compliance
- **Keyboard navigation**: All interactive elements accessible via keyboard
- **Screen reader support**: Proper ARIA labels and semantic HTML
- **Color contrast**: Minimum 4.5:1 ratio for normal text
- **Focus management**: Visible focus indicators and logical tab order

### Implementation Guidelines
- Use semantic HTML elements
- Provide alternative text for images
- Implement proper heading hierarchy
- Test with screen readers and keyboard-only navigation

## Error Handling System

### Comprehensive Error Tracking
- **Real-time monitoring**: Automatic error detection and reporting
- **Analytics dashboard**: `/admin/error-analytics` for error management
- **Automated insights**: Pattern detection and resolution recommendations
- **Performance impact**: Track how errors affect user experience

### Error Boundaries
- **Global error boundary**: `app/global-error.tsx` for critical failures
- **Route-level boundaries**: `app/error.tsx` for page-specific errors
- **Component boundaries**: Custom ErrorBoundary for component isolation

### Error Reporting
```typescript
// Report errors with context
import { reportError } from '@/lib/services/errorReporting';

reportError(error, 'javascript-error', 'high', {
  component: 'ComponentName',
  domain: 'full-stack',
  additionalData: { /* context */ }
});
```
